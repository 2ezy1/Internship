# ESP32 Auto-Registration System

## 🎯 Overview

The ESP32 devices now support **automatic registration** with the backend server. No manual database entries or hardcoded credentials are needed!

## ✨ How It Works

### First Connection (New Device)

1. **ESP32 starts up**
   - Gets its unique MAC address (e.g., `AA:BB:CC:DD:EE:FF`)
   - Checks internal flash memory (Preferences) for stored credentials
   - No credentials found → proceeds with auto-registration

2. **Connects to server**
   - WebSocket connection: `ws://server:8000/ws/esp32/connect?mac_address=AA:BB:CC:DD:EE:FF`
   - Server doesn't recognize this MAC address

3. **Server auto-registers**
   - Creates new device entry in database:
     - `device_name`: `ESP32_MASTER_CCDDEE` (last 6 chars of MAC)
     - `mac_address`: `AA:BB:CC:DD:EE:FF`
     - `ip_address`: Auto-detected from connection
     - `type`: `ESP32_Master`
     - `device_key`: Auto-generated UUID (e.g., `123e4567-e89b-12d3-a456-426614174000`)

4. **Server responds with credentials**
   ```json
   {
     "type": "registration",
     "status": "success",
     "device_id": 1,
     "device_key": "123e4567-e89b-12d3-a456-426614174000",
     "device_name": "ESP32_MASTER_CCDDEE",
     "is_new_device": true,
     "message": "Device registered successfully"
   }
   ```

5. **ESP32 saves credentials**
   - Stores `device_id`, `device_key`, and `device_name` in flash memory
   - These persist through power cycles and resets

6. **Normal operation begins**
   - ESP32 starts sending heartbeats and VFD data
   - Fully authenticated and operational

### Subsequent Connections (Existing Device)

1. **ESP32 starts up**
   - Gets MAC address
   - **Finds stored credentials** in flash memory
   - `device_id`: `1`
   - `device_key`: `123e4567-e89b-12d3-a456-426614174000`

2. **Connects with credentials**
   - WebSocket URL: `ws://server:8000/ws/esp32/connect?mac_address=AA:BB:CC:DD:EE:FF&device_id=1&device_key=123e4567...`
   - Server recognizes device immediately

3. **Server authenticates**
   - Validates credentials
   - Updates IP address if changed
   - Marks device as online

4. **Server sends confirmation**
   ```json
   {
     "type": "registration",
     "status": "success",
     "device_id": 1,
     "device_key": "123e4567-e89b-12d3-a456-426614174000",
     "device_name": "ESP32_MASTER_CCDDEE",
     "is_new_device": false,
     "message": "Device authenticated"
   }
   ```

5. **Normal operation continues**
   - No re-registration needed
   - Credentials remain the same

## 📋 ESP32 Required Data for Registration

The ESP32 automatically provides all necessary data:

### Automatically Provided:
- ✅ **MAC Address** - Unique identifier from WiFi hardware
- ✅ **IP Address** - Detected from WebSocket connection
- ✅ **Device Type** - Hardcoded as `ESP32_Master`

### Auto-Generated by Server:
- ✅ **Device Name** - Generated from MAC address (e.g., `ESP32_MASTER_CCDDEE`)
- ✅ **Device Key** - UUID generated by server for authentication

### Optional:
- ⚠️ **User Assignment** - Initially `null`, can be assigned later via frontend/API

## 🔧 Configuration

### ESP32 (config.h)

**Before (Manual Configuration):**
```cpp
#define DEVICE_ID 1
#define DEVICE_KEY "69ced61b-5521-4ef7-ab17-19a2cdf14af8"
```

**After (Auto-Registration):**
```cpp
// NO MANUAL DEVICE_ID OR DEVICE_KEY NEEDED!
// Server automatically registers using MAC address
```

Only WiFi and Server settings need configuration:
```cpp
#define WIFI_SSID "YourWiFi"
#define WIFI_PASSWORD "YourPassword"
#define SERVER_IP "192.168.1.100"
#define SERVER_PORT 8000
```

### Backend (main.py)

WebSocket endpoint signature:
```python
@app.websocket("/ws/esp32/connect")
async def websocket_esp32_handler(
    websocket: WebSocket, 
    mac_address: str,                    # Required
    device_id: Optional[int] = None,     # Optional
    device_key: Optional[str] = None,    # Optional
    db: Session = Depends(get_db)
):
```

## 💾 Data Storage

### ESP32 Flash Memory (Preferences)
Stored in namespace `esp32_device`:
- `device_id` (int)
- `device_key` (string)
- `device_name` (string)

Location: Non-volatile storage, survives:
- ✅ Power cycles
- ✅ Reboots
- ✅ Firmware updates (unless flash is erased)

### Backend Database
Device table includes:
```sql
CREATE TABLE devices (
    id SERIAL PRIMARY KEY,
    device_name VARCHAR,
    ip_address VARCHAR UNIQUE,
    type VARCHAR,
    mac_address VARCHAR,  -- NEW: Used for auto-registration
    device_key VARCHAR UNIQUE,
    is_online BOOLEAN,
    last_heartbeat TIMESTAMP,
    user_id INTEGER,
    ...
);
```

## 🚀 Benefits

1. **Zero Manual Configuration**
   - No database pre-population needed
   - No manual device_key generation
   - No editing config.h for device credentials

2. **Scalability**
   - Deploy multiple ESP32s without individual setup
   - Each automatically gets unique credentials
   - MAC address ensures no collisions

3. **Security**
   - Unique UUID device_key per device
   - Credentials stored securely in flash
   - MAC address acts as hardware identifier

4. **Reliability**
   - Credentials persist through power loss
   - Automatic re-authentication on reconnect
   - IP address updates handled automatically

## 🔐 Security Considerations

- **MAC Address Spoofing**: MAC addresses can be spoofed. Consider adding additional authentication for production (e.g., pre-shared secrets, certificate authentication)
- **Device Key Storage**: Keys stored in flash can be extracted with physical access. Consider encryption for sensitive deployments
- **User Assignment**: Initially devices are not assigned to any user. Implement user assignment workflow in frontend

## 🛠️ Troubleshooting

### Issue: ESP32 keeps re-registering

**Cause**: Preferences not being saved correctly

**Solution**:
```cpp
// Verify Preferences are initialized correctly
preferences.begin("esp32_device", false); // false = read-write
```

### Issue: Server rejects MAC address

**Cause**: MAC address format mismatch

**Solution**: Check MAC address format in logs. Should be `AA:BB:CC:DD:EE:FF`

### Issue: Credentials saved but not loading

**Cause**: Preferences namespace mismatch

**Solution**: Ensure same namespace used in `loadCredentials()` and `saveCredentials()`

### Issue: Multiple devices get same ID

**Cause**: Database constraint violation

**Solution**: Check that `mac_address` column is properly storing unique values

## 📊 Database Schema Update

If migrating from old system, ensure `mac_address` column exists:

```sql
-- Add mac_address column if it doesn't exist
ALTER TABLE devices 
ADD COLUMN IF NOT EXISTS mac_address VARCHAR;

-- Make it unique to prevent duplicates
CREATE UNIQUE INDEX IF NOT EXISTS idx_devices_mac 
ON devices(mac_address);
```
